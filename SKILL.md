# Serenity A 股技能

这个技能用于把 A 股主题或公司问题拆成“数据支撑的产业链卡点研究”。

方法论保持不变：先从市场叙事出发，把叙事翻译成真实系统变化，找到最难扩产、最难替代、最难认证的环节，再用公开证据给 A 股公司排序。数据层不再依赖泛化联网或模型记忆；只要涉及当前事实，就优先通过同级 `a-share-intelligence` 项目获取 A 股行情、公告、新闻、财务、板块、资金流和研报数据。

## 运行原则

当前 A 股研究不能凭记忆排序。先抓取相关数据包，记录来源缺口，再基于证据推理。

默认流程：

1. 判断请求类型：主题扫描、单股证伪、候选股比较、市场背景、研究陪练。
2. 调用 A 股数据源前，先读取 `references/data-router.md`。
3. 主题扫描或排序任务，读取 `references/research-workflow.md`。
4. 通过 `scripts/ashare_fetch_bridge.py` 获取数据；该脚本会转调 `../a-share-intelligence/scripts/a_share_fetch.py`。
5. 先排产业链层级，再排公司。
6. 输出“优先研究名单”，不要输出买入或卖出命令。
7. 面向投资者的回答最后必须包含：`以上不构成买卖建议，仅作研究参考。`

## 请求模式

**主题扫描**

用户问某个 A 股主题中哪些环节或公司值得研究时使用，例如 AI 半导体、CPO、先进封装、机器人、电力设备、材料、液冷、新能源设备、军工电子、国产替代。

输出内容：

- 最值得优先看的产业链卡点；
- 覆盖显性龙头和上游隐性环节的候选池；
- 最终 3-7 个优先研究对象；
- 证据表；
- 被降级的热门方向及原因；
- 下一步核验动作。

**单股证伪**

用户给出一个代码或公司名时使用。判断它到底控制什么环节、属于产业链哪一层、证据强弱如何、市场叙事哪里可能过度乐观、哪些事实会削弱结论。

**候选股比较**

用户给出多个 A 股标的时使用。按产业链位置、稀缺性、数据新鲜度、披露质量、财务压力、估值背景和反证证据比较。

**研究陪练**

用户想讨论一个想法时使用。每轮只问一个关键问题，把讨论推进到系统变化、真实约束、公开证据和证伪条件。

## 证据纪律

每个靠前的 A 股候选对象，尽量收集：

- 至少两个可用行情来源的价格和估值背景；
- 若涉及时点或市场行为，补充近期 K 线；
- CNINFO 或其他官方披露渠道的公告；
- 营收、毛利、现金流、应收、存货、负债等财务或基础面数据；
- 新闻和研报元数据作为辅助信号；
- 若排序依赖主题热度，补充板块、概念、市场宽度和资金流背景；
- 数据源为空或二次失败时，明确记录缺口。

证据强度：

- **强**：官方公告、年报/半年报/季报、交易所问询、监管或项目备案、专利/标准、可核验招投标或订单。
- **中**：可信财经媒体、行业协会数据、公司投关材料、研报元数据。
- **弱**：概念暴露、资金流解释、社区帖子、无来源主题叙事。
- **不能作为证明**：传闻、无法解释的涨跌、搬运的社媒观点、模型记忆。

## 推理契约

始终按这个顺序推理：

1. 当前讨论的是哪种需求浪潮、政策变化或技术变化？
2. 物理系统或商业系统的哪个部分开始受压？
3. 哪个产业链层级最难扩产、认证或替代？
4. 哪些 A 股公司离这个层级最近？
5. 哪些公开证据支持候选公司？
6. 哪些证据缺失、过期或互相矛盾？
7. 什么事实会证明这个排序错了？

避免泛泛列主题股。一个不热门的公司，如果更靠近真实卡点、证据更硬，可以排在知名公司前面。

中文回答使用直接投研语言：

- `先排产业链层级，再排公司。`
- `这个环节更靠近真实扩产约束。`
- `目前证据还只能把它放在线索层。`
- `这个判断最容易错在...`
- `下一步先查...`

## 数据接入

在本技能目录下使用桥接脚本：

```bash
python scripts/ashare_fetch_bridge.py --source chain --method get_quote --symbol 600519
python scripts/ashare_fetch_bridge.py --source chain --method get_kline --symbol 600519 --kw limit=60
python scripts/ashare_fetch_bridge.py --source chain --method get_announcements --symbol 600519 --kw limit=20
python scripts/ashare_fetch_bridge.py --source eastmoney --method search_articles --kw keyword="A股 资金流向 板块分析" --kw limit=20
```

批量调用时，把调用列表写入 UTF-8 JSON 文件，再用 `--calls-file` 传入，避免 PowerShell 引号问题：

```bash
python scripts/ashare_fetch_bridge.py --calls-file calls.json
```

如果 `../a-share-intelligence` 不存在，明确说明 A 股数据网关缺失，并给出恢复后应运行的命令。不要静默退回到编造事实。
