# 安全说明

`serenity-skill-ashare` 是一个本地 Agent Skill 项目，不托管服务端，也不要求默认配置密钥。

## 支持范围

可以报告：

- 脚本命令拼接导致的本地命令执行风险。
- 路径处理导致读取或执行非预期文件。
- 文档诱导用户提交私密账号、cookie、token 或非公开信息。
- 数据来源失败后静默编造结果的流程问题。
- 输出中弱化投资风险边界的规则问题。

## 不属于安全漏洞但可以作为改进建议

- 上游公开数据源短暂不可用。
- 第三方接口字段变化。
- 非交易日数据为空。
- 代理或网络环境导致的抓取失败。
- 公开新闻、公告或研报元数据本身的事实错误。

## 敏感信息

不要在 issue、PR 或示例文件中提交：

- 券商账号、交易账号或密码；
- cookie、token、API key；
- 未公开订单、客户名单、调研纪要或内幕信息；
- 个人身份证件、手机号、邮箱列表或其他隐私数据。

如果确实需要展示失败数据，请脱敏后只保留来源名、方法名、返回结构和错误摘要。

## 投资边界

本项目只处理公开资料研究流程，不提供买卖指令。即使引用 Serenity / `@aleabitoreddit` 或其他公开账号的观点，也只能作为研究线索，不能作为最终证据。任何输出都应保留：

```text
以上不构成买卖建议，仅作研究参考。
```
