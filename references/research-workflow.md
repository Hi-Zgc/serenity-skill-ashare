# A 股产业链卡点研究流程

用于 A 股主题扫描、当前机会排序和候选公司比较。

## 完成标准

当前主题扫描尽量完成：

- 覆盖至少三个产业链层级；
- 主题足够宽时，建立约 20 个 A 股候选对象的初始池；
- 最终排序前先抓取当前数据；
- 区分“层级排序”和“公司排序”；
- 每个最终候选对象都要有具体公开证据；
- 至少指出一个需要降级的热门层级或热门股票；
- 说明最快能证伪该判断的事实。

如果时间或工具受限，把回答标为初步版本，并列出缺失检查。除非用户明确要求只给计划或进入研究陪练模式，不要在列出“下一步核验”后停下等待用户；应继续执行可用的数据抓取和核验动作，直到达到可交付结论或遇到明确工具阻塞。

## 第一步：把主题翻译成系统压力

把请求转换成：

```text
需求或政策变化 -> 受压系统 -> 技术或商业要求 -> 稀缺环节
```

示例：

- AI 服务器 -> 带宽和功耗受压 -> 更高速互连、封装、散热 -> 光引擎、基板、先进封装、液冷。
- 国产半导体设备 -> 工艺自主需求 -> 晶圆厂设备替代 -> 刻蚀、薄膜沉积、清洗、量测、零部件、材料。
- 人形机器人 -> 执行器和感知密度提升 -> 精密部件和稳定量产 -> 减速器、电机、编码器、传感器、电池、加工良率。

## 第二步：拆 A 股产业链层级

使用颗粒度足够细的图谱，不要把经济属性不同的环节混成一篮子。

常见层级：

1. 终端需求和资本开支来源。
2. 系统集成商和整机厂。
3. 核心芯片、模块和控制系统。
4. 制造工艺、封装和测试。
5. 设备、量测和零部件。
6. 材料、耗材、化学品、基板和精密部件。
7. 电力、散热、基础设施和物流。
8. 软件、EDA、IP、操作系统和设计工具。

## 第三步：寻找稀缺性

多个信号叠加时，该层级排序应提高：

- 合格供应商少；
- 认证或客户导入周期慢；
- 扩产依赖专用设备、许可、工艺知识或稀缺材料；
- 客户预付款、锁产能、签长单或接受涨价；
- 财务报表开始体现收入结构变化、毛利韧性、合同负债、存货准备或资本开支落地；
- 市场仍按旧业务给公司分类，而新约束正在变强。

先写层级排序，再进入最终公司名单。

## 第四步：建立候选池

宽主题的 A 股扫描应覆盖：

- 显性龙头；
- 上游工具和材料；
- 决定良率或可靠性的部件；
- 检测、封装和量测；
- 数据源返回的板块或概念成分；
- 相邻基础设施公司；
- 需要明确降级的热门公司。

候选对象分类：

- 控制稀缺环节；
- 供应稀缺环节；
- 受益于需求但控制力有限；
- 有概念暴露但证据薄；
- 主要是交易性故事。

## 第五步：抓取并归一证据

调用路径见 `references/data-router.md`。

最终靠前公司至少尽量包含：

- 行情和基础面背景；
- 公告或定期报告证据；
- 财务或经营证据；
- 必要时补新闻和研报元数据；
- 如果排序依赖市场热度，补板块和题材背景；
- 明确的数据源缺口。

连续执行要求：

- 初步识别出待查公告、财务、新闻、研报或板块数据后，立即通过 `scripts/ashare_fetch_bridge.py` 或批量 `--calls-file` 继续抓取，不要把待查清单直接交给用户确认。
- 主题扫描候选池过大时，先自行选择最贴近真实卡点、公开披露更具体、数据可得性更高的 3-7 个对象进入核验。
- 某个数据源失败最多重试一次；第二次仍失败时记录缺口，继续查其他来源和其他候选对象。
- 年报、招股书、交易所问询回复、调研纪要等原文难以通过当前工具取得时，最多询问用户一次是否能补充材料。用户补充则纳入证据；用户不能补充或未补充时，继续用已取得数据分析，并把这些材料列为缺口。
- 只有 `../a-share-intelligence` 缺失、核心方法不可用、用户问题需要私人信息，或用户明确要求陪练讨论时，才停止继续抓取并说明原因。

硬证据关键词：

- 对 ABF、FC-BGA、IC 载板、封装载板、客户认证、批量供货、产能利用率、分产品收入等关键词，只把年报、招股书、问询回复、调研纪要、官方公告或可核验订单作为强证据。
- 如果没有上述硬证据，相关公司即使有题材暴露、新闻热度或资金流，也只能放在线索层；不要把它写成已兑现的业务结论。

财务检查重点：

- 营收增速与应收增速是否匹配；
- 存货和合同负债；
- 毛利率和费用趋势；
- 经营现金流；
- 资本开支、在建工程和产能说法；
- 再融资、质押、商誉、关联交易和摊薄。

## 第六步：排序

层级排序依据：

- 需求压力；
- 稀缺环节紧张程度；
- 供应商集中度；
- 扩产难度；
- 证据强度；
- 时间窗口；
- 估值和融资压力；
- 会计、治理和流动性风险。

然后排公司。每个最终候选对象都要回答：

- 它靠近哪个稀缺环节？
- 哪些公开证据证明或弱证明这个位置？
- 还缺什么？
- 为什么它排在显性热门公司前面或后面？
- 什么事实会迫使下调排序？

可选评分：

```bash
python scripts/ashare_bottleneck_score.py --template
python scripts/ashare_bottleneck_score.py candidate.json --format md
```

## 第七步：写出回答

先给判断，不从背景铺陈开始：

```text
先排产业链层级，再排公司。我会优先看三类：...
```

然后给出：

- 层级排序；
- 靠前候选公司表；
- 证据说明；
- 被降级的热门方向；
- 缺口和失败来源；
- 已执行核验、仍缺数据和后续待补动作；
- 固定合规句。

不要把资金流或概念板块异动当成主营业务证明。它们只能作为时点和情绪背景。
